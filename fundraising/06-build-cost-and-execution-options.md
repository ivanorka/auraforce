# Build Cost And Execution Options

This document keeps the cost conversation flexible without weakening the investor story.

## Important Framing

There are two different numbers we may discuss:

1. Lean build cost: what a focused MVP might cost if executed aggressively with an offshore or hybrid team.
2. Investment raise: what AuraForce should raise to build, pilot, iterate, cover AI/infra risk and avoid stopping before proof.

The lean route can be real, but the investor recommendation should stay more conservative.

## Fast Execution Team

- 1 product / technical lead
- 2 full-stack developers
- 1 AI / automation engineer
- 1 part-time UI/UX designer
- 1 part-time QA tester
- AI-assisted development stack

Recommended tools:

- Cursor
- Claude
- OpenAI models
- GitHub Copilot
- v0 / Lovable for rapid UI exploration
- automated testing and documentation helpers

## Phase 1 - MVP, 8-12 Weeks

Scope:

- CRM
- lead capture
- unified inbox for WhatsApp, email and chat
- calendar
- appointment booking
- AI receptionist
- AI sales assistant
- basic dashboard
- admin panel
- usage tracking

Estimated build cost:

| Team Model | Estimated Cost |
|---|---:|
| Offshore team | `$35k-$75k` |
| UK/EU team | `$80k-$150k` |
| Premium agency | `$150k-$250k+` |

AI development tools:

- coding assistants: `$100-$500/month`
- AI testing / documentation: `$100-$500/month`

Expected monthly technical running cost per 100 active clients:

- GPT / Claude usage: `$300-$2,000+`
- vector database: `$50-$300`
- hosting: `$100-$500`

## How To Present This To Investors

Best phrasing:

A very lean MVP can be built for less, but the recommended initial raise is `$350k-$500k` because AuraForce needs more than code. It needs pilot execution, integration buffer, AI/voice usage budget, security, onboarding, design, legal work and enough runway to prove paid demand.

This protects us from building a demo that runs out of budget before becoming a business.
