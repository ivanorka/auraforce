# First Product Screen Priority

The first desktop screens should prove the product in this order:

## 1. Executive Command Dashboard

Purpose:

Show that AuraForce gives the owner a clear operating view: lead speed, booked viewings, pipeline, AI employees and usage wallet.

Why first:

Investors and clients need to immediately understand that this is not a chatbot. It is a command platform.

## 2. Unified Inbox / Live Lead Desk

Purpose:

Show the actual money workflow: WhatsApp lead arrives, AI responds, qualifies, books a viewing and keeps human handoff available.

Why second:

This is the first real estate wedge in one screen.

## 3. AI Employee Studio

Purpose:

Show Lewis's key concept: AI employees have GUI control surfaces with job description, tone, permissions, scripts, escalation rules, usage limits and simulation.

Why third:

This makes AuraForce feel like a scalable platform with installable roles, not a collection of disconnected automations.

## Next Screens

Likely next desktop screens:

- Booking calendar
- Usage wallet and billing
- Real estate pipeline detail
- Knowledge base / company brain
- Admin / tenant settings

## Mobile App Direction

The first native-style mobile screens should mirror the three desktop priorities:

- Mobile Command
- Mobile Lead Desk
- Mobile AI Studio

These should use mobile-native UI patterns: status bar, concise header, large primary actions, bottom tab bar, segmented controls, compact cards and touch-friendly permission controls.
